Разверните собственный OpenClaw-ассистент и подключите его к Telegram.

Estimated Time 60-90 минут
Difficulty Level Beginner
Result AI-ассистент в Telegram

В этом модуле вы самостоятельно развернете OpenClaw и подключите его к Telegram. В результате у вас будет собственный AI-ассистент, которым можно управлять через Telegram.

Перед началом
  • Аккаунт DigitalOcean или другой VPS
  • Установленный терминал
  • Telegram
Почему отдельный сервер

Изоляция важнее удобства

OpenClaw может выполнять команды и работать с файлами. Если запускать его на личном компьютере, агент может получить доступ к вашим данным. Отдельный VPS изолирует систему и снижает риски.

Setup Steps

Пошаговая установка

Раскрывайте шаги, копируйте команды и отмечайте выполненные пункты по мере прохождения.

Цель шага

Создать отдельную машину для OpenClaw, чтобы агент не имел доступа к вашему личному компьютеру.

Что сделать
  • Откройте DigitalOcean.
  • Нажмите Create -> Droplet.
  • Выберите Ubuntu 22.04.
  • Выберите минимальный тариф.
  • После создания сохраните IP-адрес сервера.
Пример IP
134.209.xxx.xxx
🔒 Security note

OpenClaw запускается на отдельном сервере, чтобы изолировать его от вашего компьютера.

Статус
Цель шага

Убрать зависимость от входа по паролю и перейти на SSH-ключ для более безопасного доступа.

Что сделать
  • Создайте SSH-ключ локально.
  • Добавьте публичный ключ в настройки сервера.
  • Подключитесь к серверу через SSH.
  • Убедитесь, что вы видите терминал удаленного сервера.
Команды
ssh-keygen
ssh root@SERVER_IP
🔒 Security note

Подключение по паролю может быть небезопасным, поэтому для сервера лучше использовать SSH-ключ.

Статус
Цель шага

Установить последние обновления системы и закрыть известные уязвимости перед запуском OpenClaw.

Команды
apt update
apt upgrade
🔒 Security note

Обновления закрывают известные уязвимости системы и уменьшают риск эксплуатации старых пакетов.

Статус
Цель шага

Подготовить среду сервера для клонирования репозитория и запуска OpenClaw в контейнере.

Команды
apt install git docker.io
🔒 Security note

Контейнеризация создает дополнительный уровень изоляции и делает окружение OpenClaw более контролируемым.

Статус
Цель шага

Подготовить отдельную директорию для OpenClaw и workspace, к которому агент будет иметь ограниченный доступ.

Команды
mkdir /home/openclaw
mkdir /home/openclaw/workspace
cd /home/openclaw
🔒 Security note

Workspace ограничивает область доступа агента и снижает вероятность того, что OpenClaw затронет лишние файлы.

Статус
Цель шага

Развернуть OpenClaw на сервере и запустить основной сервис в контейнере.

Команды
git clone https://github.com/openclaw/openclaw
cd openclaw
docker compose up
🔒 Security note

OpenClaw должен работать в контейнере на отдельном сервере, а не на вашем ноутбуке или рабочей машине.

Статус
Цель шага

Подключить модель, чтобы OpenClaw мог отвечать на сообщения и выполнять команды.

Пример переменной
OPENAI_API_KEY=your_key_here
🔒 Security note

API-ключи нельзя хранить прямо в коде или публиковать в репозитории. Используйте переменные окружения.

Статус
Цель шага

Подготовить Telegram-интерфейс, через который вы будете общаться с OpenClaw.

Что сделать
  • Откройте Telegram и найдите BotFather.
  • Введите команду /newbot.
  • Укажите имя бота и username.
  • Сохраните выданный token.
Команда и пример токена
/newbot
123456:ABCDEF...
🔒 Security note

Bot token дает полный доступ к управлению ботом, поэтому храните его так же аккуратно, как и API-ключи.

Статус
Цель шага

Связать Telegram-бота с OpenClaw, чтобы агент мог принимать и обрабатывать ваши сообщения.

Что сделать
  • Добавьте bot token в конфигурацию OpenClaw.
  • Перезапустите сервис, если это требуется конфигурацией.
  • Отправьте боту сообщение hello.
Тестовое сообщение
hello
🔒 Security note

После подключения проверьте, кто может писать вашему боту, и не оставляйте интерфейс полностью открытым без контроля доступа.

Статус
Цель шага

Перед использованием убедиться, что OpenClaw работает в изолированной среде и не раскрывает секреты.

Что проверить
  • OpenClaw работает на отдельном сервере.
  • Доступ к серверу только через SSH.
  • Агент работает внутри workspace.
  • API-ключи не лежат в коде.
🔒 Security note

Эти базовые меры уменьшают риск утечки данных, лишнего доступа к системе и атак вроде prompt injection.

Цель шага

Проверить конечный результат: бот принимает сообщения и отвечает через OpenClaw.

Примеры сообщений
hello
what is my schedule today
🔒 Security note

Даже после успешного запуска продолжайте тестировать бота на простых командах, прежде чем подключать реальные рабочие сценарии.

Quick Notes

Что важно помнить

Отдельный сервер

Не запускайте OpenClaw на своем личном компьютере. Изоляция важнее удобства.

Секреты вне кода

API-ключи и bot token должны жить в конфигурации, а не в репозитории.

Тестируйте поэтапно

Проверяйте сервер, SSH, OpenClaw и Telegram отдельно, а не все сразу.
